# 任务二:植被/水体分类(进入 GIS+AI)

> 前置:做完任务一,养成了"必须验证、必须看懂"的习惯,再做这个。

## 这次的新难点

任务一里,对错你能算、能看。**这次不行。** 分类模型给你一张结果图,它可能 90% 对、也可能在系统性地犯错,而图看起来都差不多。所以这个任务真正要练的,是**当你没法肉眼判断对错时,怎么相信(或不相信)一个模型**。

## 玩法(和任务一一致,外加两条)

1. 规格先行,代码让 AI 写,但你要能逐段解释。
2. 每一步都要验证——这次"验证"指的是用对的方式评估模型,而不是看一眼结果图。
3. **新增:不许只看 "准确率(accuracy)" 这一个数字就下结论。**
4. **新增:你要能回答一个问题——"你怎么保证你的测试集没在偷看训练集?"**

## 你的任务

> "这景影像里有植被和水体,我想训练一个模型自动把它们分出来。"

导师给你 `scene.tif`(4 波段:Red/NIR/Green/SWIR)和 `labels.tif`(地面真值:0=其他, 1=植被, 2=水体)。

## 第一步:填规格(填完拿给导师过一遍)

```markdown
# 目标
# 输入(几个波段?标签几类?各类占比是多少?——去统计一下)
# 方法(用什么模型?为什么?特征是什么?要不要加 NDVI 之类的指数?)
# 训练/测试怎么切分(⚠️ 想清楚:像素之间是独立的吗?)
# 评估指标(只看 accuracy 够吗?还要看什么?)
# 验收标准(可执行)
# 非目标
```

## 第二步:做完后必做的自检(过不了别交)

1. **先算多数类基线**:统计各类占比,如果某类占 85%,那"全猜它"就有 85% 准确率。你的模型有没有明显超过这个基线?没有 = 没学到东西。
2. **不能只看 accuracy**:打印**混淆矩阵**和**每一类的 precision/recall**。看看占比小的那一类(水体)到底分对了多少——总体准确率会把它的失败藏起来。
3. **检验数据泄漏**(这个任务最重要的一关):你是怎么切训练/测试集的?如果是"所有像素随机打乱再切",请警惕——影像里地物是成片的,相邻像素几乎一模一样,随机切分会让测试集里全是训练集的"近邻",准确率虚高到接近完美。**问自己:换一种按空间区块切分的方式,准确率会不会掉下来?** 去试,把两种切分的结果都报出来。
4. **空间合理性**:把预测结果作为一张图叠回原影像,水体的形状是不是连成片、符合地物分布,还是碎成椒盐噪声。

## 第三步:交付

交规格、代码、以及一份**评估报告**:多数类基线、混淆矩阵、各类 precision/recall、两种切分方式下的准确率对比。导师会重点和你讨论第 3 条。

这个任务做完,你要能用一句话说清:**为什么一个准确率 99% 的模型,可能其实毫无用处。** 答得出来,你就摸到 GIS+AI 的门了。
